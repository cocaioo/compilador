4. ESPECIFICAÇÃO

Java Script Simplificado (JSS) é uma linguagem baseada na sintaxe do JavaScript, porém projetada com
tipagem forte e estrutura simplificada para ser compilada. Essa linguagem é sensível (case-sensitive) em
relação a letras maiúsculas e minúsculas nos nomes das variáveis, funções e palavras reservadas. (i.e.,
“var”, “VAR” e “VaR” representam identificadores diferentes).

A linguagem JSS deve suportar declarações e chamadas de funções, incluindo chamadas recursivas. A
linguagem possui funções built-in, utilizadas para realizar entrada e saída de dados via console do sistema
operacional, que serão as únicas formas de se comunicar com o mundo exterior.

Nenhuma característica de outra linguagem deve ser reconhecida pelo compilador.

4.1 Regras para uso de variáveis, constantes e seu escopo

Variáveis podem ter identificadores de qualquer tamanho maior que zero e devem obrigatoriamente
começar com uma letra ou underscore. Após isso, podem conter qualquer letra, digito ou o símbolo
underscore (_). Os identificadores não podem ter o mesmo nome de uma palavra reservada da linguagem
(if, while, etc.) nas sentenças descritas nas próximas seções.

O escopo define a visibilidade de um identificador (variável ou constante) onde pode ser acessado.

     • Escopo Global: Identificadores declarados fora de qualquer função ou bloco ({ ... }) são
          considerados globais e acessíveis de qualquer parte do programa.

     • Escopo de Bloco (Local): let e const possuem escopo de bloco. Isso significa que são visíveis
          apenas dentro do bloco ({ ... }) em que foram declaradas (incluindo blocos de funções, if, while,
          etc.) e em sub-blocos.

     • Re-declaração: Não é permitido declarar novamente um identificador usando let ou const no
          mesmo escopo.
4.2 Tipos de variáveis e sentenças de atribuições
A linguagem JSS possui tipos os primitivos e tipos derivados a seguir.

4.2.1 Tipos Primitivos
     • inteiros (int): Números inteiros (ex: 1, -5). Valor padrão: 0;
     • reais (real): Números de ponto flutuante (ex: 1.5, -10.8E2). Valor padrão: 0.0;
     • strings (str): Sequências de texto (ex: "Olá\n"). Valor padrão: string vazia;
     • booleanos (bool): Valores lógicos (true ou false). Valor padrão: false;

Sentenças de declaração de primitivo:
let <tipo> <identificador>;
let <tipo> <identificador> = <expressão>;
let <tipo> <lista_de_identificadores>;

Exemplo:
let int i1;
let int i2 = 10;
let int i3, i4, i5;

Sentenças de atribuição para primitivos:
<identificador> = <expressão>;

Exemplo:
i1 = 1;
i2 = 2 + 3;
i3 = 10;
i4 = 100;
i5 = 1000;

4.2.2 Tipos Derivados
     • Vetor: uma coleção ordenada de valores do mesmo tipo, acessada através de um índice numérico.
          Apenas na declaração é possível atribuir uma lista de valores a um vetor. Vetores constantes não
          podem ter seus valores alterados. Quando não iniciado, cada elemento tem o valor padrão do tipo.

Sentenças de declaração de vetor:
let <tipo> [dimensão] <identificador>;
let <tipo> [dimensão] <identificador> = <lista_de_expressões>;

Exemplo:
let int [3] l1;
let int [3] l2 = [0+1 ,1+2 ,2+3];
const str[3] nomes = ["Ana", "Bruno", "Carlos"];

4.3 Operadores

A linguagem deve suportar os operadores aritméticos, lógicos e de atribuição unário ou binário abaixo:

     • ! negação, inverte o valor booleano ao qual foi aplicada – unário;
     • + mais unário, mantém o valor inteiro ou real ao qual foi aplicada – unário;
     • - menos unário, inverte o valor inteiro ou real ao qual foi aplicada – unário;
     • + soma ou concatenação de strings – binário;
     • - subtração – binário;
     • multiplicação – binário;
     • / divisão inteira e real – binário;
     • % resto de uma divisão de inteiros – binário;
     • ** exponenciação de inteiros – binário;
     • == igual – binário;
     • != diferente – binário;
     • > maior que – binário;
     • >= maior ou igual – binário;
     • < menor que – binário;
     • <= menor ou igual – binário;
     • && E lógico (conjunção) – binário;
     • || OU lógico (disjunção) – binário;
     • ++ incremento de uma unidade (pré-fixado) – unário;
     • -- decremento de uma unidade (pré-fixado) – unário;
     • = atribuição, pode ser simples ou composta realizando uma operação simultaneamente (exemplo:

          +=, -=) – binário;

A precedência de operadores segue os seguintes critérios

I. Quando dois operadores diferentes forem utilizados ao mesmo tempo, o operador de maior precedência
será avaliado primeiro;

II. Operadores de mesma precedência serão avaliados da esquerda para a direita, exceto atribuições e
exponenciações;

III. Operadores aritméticos binários fazem conversão implícita para real caso um dos operandos seja real,
assim como o operador de concatenação faz para string quando um dos operandos é string;

IV. É possível forçar a precedência de uma operação, colocando-a entre parênteses;

V. Os operadores lógicos binários operam em curto-circuito, ou seja, o segundo operando não será avaliado
se o resultado da expressão puder ser determinado apenas pelo primeiro.

4.4 Funções

Funções são declaradas com a palavra chave function e têm um nome, uma lista de parâmetros e um tipo
de retorno, que não pode ser vetor. No caso em que não há retorno a função deve ser declarada com a
palavra chave void e não precisam obrigatoriamente de um retorno com o comando return.

Funções tem escopo global (não pode haver declaração de funções aninhadas) e não deve ter o mesmo
nome que outra função, variável ou constante já declarada. Adicionalmente, funções podem chamar
outras funções ou ser chamadas recursivamente.

Sentença de função:

function <tipo > <nome_da_função > ( <lista_de_parâmetros > ) {
          ...
          return <expressão >;

}

Exemplo:

function real media3(real nota1 , real nota2 , int nota3 ) {
          let real media = (n1+n2+n3)/3;
          return media ;

}

4.5 Funções Nativas
O compilador deverá reconhecer previamente um conjunto de funções nativas.

     • Função input
Requisita a entrada de números inteiros, reais e strings a partir do teclado (entrada padrão). O valor
fornecido é armazenado na variável usada na chamada do comando input. O comando de leitura
interrompe a execução do programa até que o usuário digite um valor (o valor é confirmado após pressionar
a tecla “<Enter>”).
Sentença de entrada padrão:
input(<lista de variáveis>);

     • Função console.log
Envia para o console (saída padrão) números inteiros, reais e strings e a cada chamada também é enviada
uma quebra de linha, exceto entre os itens de uma lista de expressões. Essa lista deve ser separada por
vírgulas. A saída dos itens da lista são separados por um espaço no console.
Sentença de saída padrão:
console.log(<lista_de_expressões>);

     • Funções de Conversão (Casting)
Permitem a conversão explícita entre os tipos primitivos suportados (str, int, real, bool). Qualquer tipo
primitivo pode ser convertido para string, ao passo que numéricos e booleanos (int, real e bool) são
intercambiáveis.

Sentença de conversão:
int(<expressão>): Converte para inteiro. Reais são truncados. Booleans: true → 1, false → 0.
real(<expressão>): Converte para ponto flutuante. Booleans: true → 1.0, false → 0.0.
bool(<expressão>): Converte para booleano. Numéricos: 0 ou 0.0 → false, qualquer outro valor → true.
str(<expressão>): Converte o valor para sua representação textual.

Exemplo:

input (a);

input (b, c, d);
